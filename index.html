Website Migration Instructions

This document lists the steps that need to be taken to copy valuelinksoftware.com website from the preview environment to the live environment.

Step 1: Backup Preview Site

  1. Login to wp-admin of preview2.valuelinksoftware.com
  2. Take a backup of the preview2.valuelinksoftware.com database using the wp migrate plugin.
  3. Login to wp-admin and go to Plugins > wp-migrate.
  4. Enable the plugin if it is not activated.
  5. Click on "Export Database," replace the URL and path with the live domain URL and path, and then click "Export Database."
  6. Extract the downloaded database.
  7. Rename the database with a specific version number, e.g., preview2.valuelinksoftware.com-1.1.sql.
  8. Zip and download site content.
  9. SSH into preview2.valuelinksoftware.com.
  10. Change directory to /var/www/preview2.valuelinksoftware.com.
  11. Create a tar.gz file: tar â€“czf preview2.valuelinksoftware.com.tar.gz *
  12. Download the zip file to your local machine, and rename it to preview2.valuelinksoftware.com-1.1.

Step 2: Create a New Database in Azure

  1. Log in to Azure and switch to the Xoopahoutlook.onmicrosoft.com subscription.
  2. Navigate to Azure Database for MySQL servers > xpsites-eus-mysql-server > Databases, and click "Add."
  3. Create a new database with an incremental version number, e.g., valuelinksoftware-1-1.

Step 3: Upload the Database to Live Server

  1. Login to the live valuelinksoftware Linux VM using an FTP client (e.g., FileZilla).
  2. Navigate to the /tmp directory.
